Accordion Component

\n\n

\n A vertically stacked set of interactive headings that each reveal a section of content. Fully accessible and animated.\n

\n
\n\n
\n
\n

Overview

\n \n

\n The Accordion component is a vertically stacked set of interactive headings that each reveal a section of content. \n Each accordion item can be expanded or collapsed to reveal the content associated with it. Only one accordion item \n can be open at a time (single expansion pattern).\n

Hyperscript Functionality

\n

\n The accordion uses Hyperscript for its interactive behavior. Each trigger button has a script that:\n

\n \n
    \n
  • Checks if the accordion is currently open or closed
    • \n
  • Toggles the aria-expanded and data-state attributes
    • \n
  • Adds or removes the .hidden class from the content panel
    • \n
  • Closes any other open accordion items
    • \n
\n \n
\n 
_=\"on click \n  if @aria-expanded equals 'true'\n    set @aria-expanded to 'false' \n    set @data-state to 'closed'\n    add .hidden to #accordion-content-1\n  else\n    set @aria-expanded to 'true' \n    set @data-state to 'open'\n    remove .hidden from #accordion-content-1\n    for otherTrigger in <button:not(#accordion-trigger-1)/>\n      set otherTrigger@aria-expanded to 'false'\n      set otherTrigger@data-state to 'closed'\n      add .hidden to <div[data-state=\\\"open\\\"] .overflow-hidden/>\n    end\n  end\"

Accessibility

\n \n
\n

\n The accordion component follows the WAI-ARIA Authoring Practices Guide for accordions. Key accessibility features include:\n

\n \n
\n
\n

Keyboard Navigation

\n
    \n
  • Tab - Moves focus between accordion headers
    • \n
  • Enter or Space - Expands/collapses the focused accordion header
    • \n
\n
\n \n
\n

Screen Reader Support

\n
    \n
  • Proper ARIA roles and attributes
    • \n
  • Content visibility announced correctly
    • \n
  • Focus management
    • \n
  • Semantic HTML structure
    • \n

Best Practices

\n \n
\n
\n

Implementation Guidelines

\n
    \n
  • Ensure unique IDs for each trigger and content pair
    • \n
  • Maintain proper ARIA attributes for accessibility
    • \n
  • Use semantic HTML elements (button for triggers)
    • \n
  • Keep content concise and scannable
    • \n
  • Test with keyboard navigation
    • \n
\n
\n \n
\n

Performance Considerations

\n
    \n
  • Use .hidden class instead of removing elements from DOM
    • \n
  • Limit the number of accordion items on a single page
    • \n
  • Lazy load content for large accordions
    • \n
  • Use CSS transitions for smooth animations
    • \n
