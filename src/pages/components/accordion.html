Overview

The Accordion component is a vertically stacked set of interactive headings that each reveal a section of content. Each accordion item can be expanded or collapsed to reveal the content associated with it. Only one accordion item can be open at a time (single expansion pattern).

Use Cases

  • FAQ sections
  • Product specifications
  • Settings panels
  • Navigation menus
  • Terms and conditions

Features

  • Fully accessible (WAI-ARIA compliant)
  • Keyboard navigation support
  • Smooth animations
  • Responsive design
  • Single expansion pattern
  • Customizable styling

Getting Started

Basic Structure

The accordion component follows a specific HTML structure: 

<div data-orientation="vertical">
  <div data-state="closed" 
       data-orientation="vertical" 
       class="border-b">
    <h3 data-orientation="vertical" 
        data-state="closed" 
        class="flex">
      <button
        type="button"
        aria-controls="content-id"
        aria-expanded="false"
        data-state="closed"
        data-orientation="vertical"
        id="trigger-id"
        class="flex flex-1 items-center 
               justify-between py-4
               text-sm font-medium
               transition-all 
               hover:underline">
        Accordion Title
        <svg class="h-4 w-4 shrink-0 
                     text-muted-foreground 
                     transition-transform 
                     duration-200">
          <!-- Chevron icon -->
        </svg>
      </button>
    </h3>
    <div
      data-state="closed"
      id="content-id"
      role="region"
      aria-labelledby="trigger-id"
      data-orientation="vertical"
      class="overflow-hidden 
             text-sm
             transition-all 
             duration-300 
             ease-in-out 
             hidden">
      <div class="pb-4 pt-0">
        Content goes here.
      </div>
    </div>
  </div>
</div>

Key Attributes

Attribute Element Description
data-state Item, Trigger, Content "open" or "closed"
data-orientation All elements Set to "vertical"
aria-controls Trigger Button References content panel ID
aria-expanded Trigger Button Indicates visibility

Interactivity

Hyperscript Logic

The accordion uses Hyperscript for interactions:

  • Toggles aria-expanded and data-state
  • Adds/removes .hidden class
  • Adds/removes .rotate-180 class to icon
  • Closes other accordion items
_="on click 
  if @aria-expanded equals 'true'
    set @aria-expanded to 'false' 
    set @data-state to 'closed'
    add .hidden to #content-id
    remove .rotate-180 from 
  else
    set @aria-expanded to 'true' 
    set @data-state to 'open'
    remove .hidden from #content-id
    add .rotate-180 to 
    for otherTrigger in 
      <button:not(#trigger-id)/>
      set otherTrigger@aria-expanded to 'false'
      set otherTrigger@data-state to 'closed'
      add .hidden to 
        <div[id^='accordion-content-']:
        not(#content-id)/>
      remove .rotate-180 from 
    end
  end"

Icon Rotation

The chevron icon rotates 180 degrees when the accordion is open: 

<button>
  <svg class="transition-transform duration-200">
    <!-- Chevron icon -->
  </svg>
</button>

When the accordion is open, we add the .rotate-180 class to the SVG: 

<button>
  <svg class="transition-transform duration-200 rotate-180">
    <!-- Chevron icon -->
  </svg>
</button>

Accessibility

The accordion follows WAI-ARIA practices for accessibility:

Keyboard Navigation

  • Tab - Move between headers
  • Enter or Space - Toggle header

Screen Reader

  • Proper ARIA roles and attributes
  • Content visibility announced
  • Focus management
  • Semantic HTML structure
