Accordion

A vertically stacked set of interactive headings that each reveal a section of content.

Yes. It adheres to the WAI-ARIA design pattern.

Yes. It comes with default styles that match the other components' aesthetics.

Yes. It's animated by default, but you can customize the animation as needed.

Implementation

Basic Structure

The accordion component consists of a container with multiple accordion items. Each item has a trigger button and a content panel. 

<div data-orientation="vertical">
  <div data-state="closed" data-orientation="vertical" class="border-b">
    <h3 data-orientation="vertical" data-state="closed" class="flex">
      <button
        type="button"
        aria-controls="accordion-content-1"
        aria-expanded="false"
        data-state="closed"
        data-orientation="vertical"
        id="accordion-trigger-1"
        class="flex flex-1 items-center justify-between py-4 font-medium transition-all hover:underline [&[data-state=open]>svg]:rotate-180">
        Is it accessible?
        <!-- Icon -->
      </button>
    </h3>
    <div
      data-state="closed"
      id="accordion-content-1"
      role="region"
      aria-labelledby="accordion-trigger-1"
      data-orientation="vertical"
      class="overflow-hidden text-sm hidden">
      <div class="pb-4 pt-0">
        Yes. It adheres to the WAI-ARIA design pattern.
      </div>
    </div>
  </div>
</div>

Functionality with Hyperscript

The accordion uses Hyperscript to handle the interactive behavior. Each trigger button has a script that toggles its content panel. 

_="on click 
  if @aria-expanded equals 'true'
    set @aria-expanded to 'false' 
    set @data-state to 'closed'
    add .hidden to #accordion-content-1
  else
    set @aria-expanded to 'true' 
    set @data-state to 'open'
    remove .hidden from #accordion-content-1
    for otherTrigger in <button:not(#accordion-trigger-1)/>
      set otherTrigger@aria-expanded to 'false'
      set otherTrigger@data-state to 'closed'
      add .hidden to <div[data-state="open"] .overflow-hidden/>
    end
  end"

Accessibility Features

  • aria-controls attribute on the trigger button that points to the content panel
  • aria-expanded attribute that indicates whether the content is visible
  • role="region" on the content panel
  • aria-labelledby attribute on the content panel that points to the trigger button
  • Keyboard navigation support

Animation

The accordion uses Tailwind CSS transitions for smooth animations. The chevron icon rotates 180 degrees when the accordion is open. 

<svg class="h-4 w-4 shrink-0 text-muted-foreground transition-transform duration-200">
  <!-- Icon path -->
</svg>
.transition-transform.duration-200 {
  transition-property: transform;
  transition-duration: 200ms;
}

[&[data-state=open]>svg]:rotate-180 {
  &[data-state="open"] > svg {
    transform: rotate(180deg);
  }
}

Usage Example

To use the accordion component, simply copy the structure and modify the content as needed. You'll need to ensure:

  • Each trigger button has a unique ID
  • Each content panel has a unique ID that matches the aria-controls attribute
  • Update the Hyperscript to reference the correct IDs
  • Maintain the data attributes for proper functionality
